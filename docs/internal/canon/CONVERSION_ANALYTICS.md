# 1️⃣ ForgePilot Starter → Pro Conversion Analytics

### 🎯 Goal:

Identify when an explorer becomes a builder.

---

## Activation Metrics

* First blueprint generated
* Second blueprint generated (repeat usage = real intent)
* Blueprint export event
* Governance module viewed

Activation threshold:

> 2+ blueprint sessions within 14 days

---

## Engagement Metrics

* Average blueprint depth score (how complex the ideas are)
* Time spent in architecture section
* Revenue model refinements
* Revisions per blueprint

Signal:
If user iterates technical sections more than 2 times → they’re serious.

---

## Maturity Signals (Upgrade Indicators)

Trigger upgrade prompts when:

* User exports blueprint
* User requests implementation details
* User asks about deployment
* User asks about AI model integration
* Blueprint complexity score exceeds threshold
* Governance section read completely

Upgrade logic example:

If:

* 3+ sessions
* 1+ export
* Technical depth score > medium

→ Show “Build With Structure” upgrade banner

---

## Risk Signals (Churn Risk)

* Only 1 session
* No blueprint completion
* High bounce from revenue section
* No return in 7 days

Remedy:
Send “Refine your idea” reactivation email.

# 2️⃣ ForgePilot Pro → OMEGA Conversion Analytics

### 🎯 Goal:

Detect when planning becomes implementation.

---

## Activation Metrics

* Downloaded dev-ready architecture doc
* Accessed deployment checklist
* Viewed AI architecture section
* Governance implementation guidance clicked

---

## Maturity Signals

High-confidence OMEGA candidates:

* Blueprint includes AI workflows
* Multi-tenant architecture mentioned
* Automated actions specified
* Payment flows defined
* User asks about scaling
* Traffic projections > 100 users
* “AI product” category detected

If 3+ of these are true → Flag as OMEGA candidate.

---

## Behavioral Signals

* Time spent on technical stack sections
* Number of AI feature revisions
* Mentions of “API,” “agents,” “automation”
* Blueprint marked as “launching soon”

---

## Upgrade Trigger UI

Instead of banner spam:

Inline contextual prompt inside blueprint:

> “This product includes AI-powered automation. Consider deploying with OMEGA for deterministic execution and governance.”

Subtle. Precise. No hype.

---

## Risk Signals

* Repeated architecture changes
* User asking how to self-host AI infra
* User expressing concern about logging/auditing

These are governance pain indicators.
