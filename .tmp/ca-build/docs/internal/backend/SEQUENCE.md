# ForgePilot Backend Sequence Diagrams

## 1. Create Campaign Flow

```mermaid
sequenceDiagram
    participant Client
    participant API as forgepilot-api
    participant Fed as Federation Core
    participant OMEGA as OMEGA Pantheon

    Client->>API: POST /api/v1/campaigns
    Note over Client,API: {business_idea, target_audience, tenant_id, actor_id}

    API->>API: Generate correlation_id
    API->>API: Validate request (Pydantic)

    API->>Fed: POST /conversations
    Note over API,Fed: Mission template with OMEGA/KEON workflow

    Fed->>OMEGA: Initialize conversation
    OMEGA->>Fed: conversation_id

    Fed->>API: 201 Created {conversation_id}

    API->>API: Store campaign_id → conversation_id mapping

    API->>Client: 201 Created
    Note over API,Client: {campaign_id, status: "in_progress", correlation_id}
```

## 2. Stream Progress Flow

```mermaid
sequenceDiagram
    participant Client
    participant API as forgepilot-api
    participant Fed as Federation Core
    participant Agents as MCP Agents

    Client->>API: GET /api/v1/campaigns/{id}

    API->>API: Lookup conversation_id

    API->>Fed: GET /conversations/{conversation_id}

    Fed->>Agents: Query agent states
    Agents->>Fed: Progress updates

    Fed->>API: Conversation state
    Note over Fed,API: {messages, artifacts, state}

    API->>API: Map to campaign status
    Note over API: Extract phase, percentage, last_message

    API->>Client: 200 OK
    Note over API,Client: {status, progress, updated_at}
```

## 3. Fetch Artifacts Flow

```mermaid
sequenceDiagram
    participant Client
    participant API as forgepilot-api
    participant Fed as Federation Core
    participant Storage as Artifact Storage

    Client->>API: GET /api/v1/campaigns/{id}/artifacts

    API->>API: Lookup conversation_id

    API->>Fed: GET /conversations/{conversation_id}

    Fed->>Fed: Check conversation state

    alt Conversation not completed
        Fed->>API: 200 OK {state: "in_progress"}
        API->>Client: 409 Conflict
        Note over API,Client: Campaign not completed
    else Conversation completed
        Fed->>Storage: Fetch artifacts
        Storage->>Fed: Artifact data

        Fed->>API: 200 OK {artifacts}
        Note over Fed,API: {brand_name, tagline, domains, guidelines, legal}

        API->>Client: 200 OK
        Note over API,Client: {campaign_id, artifacts, generated_at}
    end
```

## 4. Health Check Flow

```mermaid
sequenceDiagram
    participant LB as Load Balancer
    participant API as forgepilot-api
    participant Fed as Federation Core

    LB->>API: GET /health (liveness)
    API->>LB: 200 OK {status: "ok"}
    Note over API: Always returns 200 if process alive

    LB->>API: GET /ready (readiness)

    API->>Fed: GET /health

    alt Federation available
        Fed->>API: 200 OK
        API->>LB: 200 OK {status: "ready", federation_available: true}
    else Federation unavailable
        Fed--xAPI: Timeout/Error
        API->>LB: 503 Unavailable
        Note over API,LB: {status: "unavailable", federation_available: false}
    end
```

## 5. Error Handling Flow

```mermaid
sequenceDiagram
    participant Client
    participant API as forgepilot-api
    participant Fed as Federation Core

    Client->>API: POST /api/v1/campaigns
    Note over Client,API: Invalid request payload

    API->>API: Pydantic validation

    alt Validation failure
        API->>Client: 400 Bad Request
        Note over API,Client: {error: {code, message, correlation_id}}
    else Federation unavailable
        API->>Fed: POST /conversations
        Fed--xAPI: Connection refused
        API->>Client: 502 Bad Gateway
        Note over API,Client: {error: {code: "FEDERATION_UNAVAILABLE"}}
    else Federation error
        API->>Fed: POST /conversations
        Fed->>API: 500 Internal Error
        API->>Client: 502 Bad Gateway
        Note over API,Client: {error: {code: "FEDERATION_ERROR"}}
    end
```

## Key Patterns

### Correlation ID Flow
- Generated by API if not provided
- Propagated to Federation Core
- Returned in all responses
- Used for distributed tracing

### State Mapping
```
Federation State → Campaign Status
----------------------------------
queued         → queued
active         → in_progress
completed      → completed
failed         → failed
```

### Error Codes
- `VALIDATION_ERROR` - Request validation failed
- `CAMPAIGN_NOT_FOUND` - Campaign ID not found
- `CAMPAIGN_NOT_COMPLETED` - Artifacts requested before completion
- `FEDERATION_UNAVAILABLE` - Cannot reach Federation Core
- `FEDERATION_ERROR` - Federation Core returned error
- `INTERNAL_ERROR` - Unexpected API error

## Architecture Principles

1. **Thin Translation Layer**: API only translates HTTP ↔ Federation protocol
2. **No State Management**: Campaign state lives in Federation Core
3. **Fail Fast**: Return 503 immediately if Federation unavailable
4. **Correlation Tracking**: Every request gets correlation_id
5. **Structured Errors**: Consistent error format across all endpoints
